🔍 搜索方法对比

1. Gemini Brain 技能 需要API密钥

功能: 智能问答 + 搜索增强 + 代码生成 + 文档分析

需求: Google Gemini API 密钥

状态: 技能已安装，等待API密钥配置

# 使用示例（需要API密钥）
openclaw skill gemini-brain "你的问题"
openclaw skill gemini-brain --search "搜索内容"

2. web_search 工具 需要API密钥

功能: 网页搜索（使用Brave Search API）

需求: Brave Search API 密钥

状态: 需要配置API密钥

# 错误信息示例
{"error": "missing_brave_api_key",
"message": "web_search needs a Brave Search API key..."}

3. web_fetch 工具 ✅ 无需API

功能: 获取特定URL的网页内容

需求: 无需API密钥

状态: ✅ 正常工作

# 使用示例
web_fetch("https://example.com")
web_fetch("https://httpbin.org/html", maxChars=500)

4. browser 工具 ✅ 无需API

功能: 控制浏览器访问网站

需求: 无需API密钥，需要浏览器扩展

状态: ✅ 可用（需要配置）

# 使用示例
browser(action="open", targetUrl="https://example.com")
browser(action="snapshot", targetUrl="https://google.com")

🚀 推荐的工作流程

方案A: 配置API密钥（完整功能）

  1. 获取 Gemini API 密钥（用于智能搜索）
  2. 获取 Brave Search API 密钥（用于网页搜索）
  3. 配置环境变量或配置文件
  4. 享受完整的搜索功能

方案B: 无API工作流程（当前状态）

  1. 使用 web_fetch 获取特定网站内容
  2. 使用 browser 工具进行可视化搜索
  3. 结合本地知识库和文件搜索
  4. 手动收集信息后进行分析

方案C: 混合模式

  1. 对已知信息源使用 web_fetch
  2. 对需要交互的网站使用 browser
  3. 对复杂分析使用本地AI模型（如当前使用的deepseek）
  4. 逐步添加API功能
